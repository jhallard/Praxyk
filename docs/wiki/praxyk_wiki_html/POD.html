One of Praxyk's primary features is its Prediction-on-Demand capability. Various pre-trained machine learning services are provided out of the box. All of this functionality is implemented inside the containers in a C++ library and passed up to Python through SWIG. Internal to the containers, each of these functions can be accessed as follows:

Optical Character Recognition (OCR)

The OCR functionality uses the Tesseract OCR library, with pre-trained data combined with training from our own handwriting. To access it, run Python and run the following:

import praxyk
img_string = praxyk.get_string_from_image("img.jpg")

A Python RuntimeError will be thrown if the file does not exist or if the underlying libraries do not recognize the file as an image.

A multi-threaded version of this function exists as well and can be used as follows.

import praxyk
img_vec = praxyk.string_vector()
img_vec.push_back("img1.jpg")
img_vec.push_back("img2.jpg")
img_vec.push_back("img3.jpg")
img_vec.push_back("img4.jpg")
img_strings = praxyk.get_strings_from_images(img_vec)

Note that this function takes in a praxyk.string_vector() and not a standard Python list.

Spam Detection

Spam detection is done through the MLPack library. The exposed functionality takes in a string and returns the change (0.0-1.0) that the string is a spam message. To access it, run Python and run the following:

import praxyk
spam_chance = praxyk.get_spam_chance(msg)
