The queueing system for Praxyk is based off of RQ, available here.

Concept

The queue would take in requests as they come, and distribute them to available workers in containers. Enqueueing a request is as simple as:

    queue.enqueue(func_call, arg1, arg2,...)

These requests will then be picked up by the workers, which will execute the desired function call.

Build and sample execution

The prerequisite to RQ is the Redis server. To install:

    git checkout -b master_queueing
    cd queuing/redis
    make CC=clang
    sudo make install
    cd ..

Following this, open thee new terminals. In one, run redis-server (you can daemonize this later). In the others, run rqworker. THese two will listen for jobs. Now run queue.py. This will simply run a function called hello() 10,000, which prints "Hello, world!".

You can see the output in the two workers' logs onscreen.

This is currently the most rudimentary implementation of the queueing system. More details to come as features are implemented.
